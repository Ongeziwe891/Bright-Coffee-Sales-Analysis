# Bright-Coffee-Sales-Analysis
📌 **Project Overview**

This project analyzes the “Bright Coffee Shop Sales” dataset, which contains daily transactional data from a coffee shop.

The analysis was conducted to support the company’s new CEO in making data-driven decisions to increase revenue and improve product performance.

As a Junior Data Analyst, the goal of this project is to extract meaningful insights using SQL, data analysis, and data visualization techniques.

🎯 **Objectives**

The project aims to answer the following key business questions:

Which products generate the most revenue
What time of day the store performs best
Sales trends across different products and time intervals
How sales performance can be improved

🗂️ **Dataset Description**

The dataset includes transactional-level data with the following key fields:

transaction_id – Unique transaction identifier
transaction_date – Date of transaction
transaction_time – Time of purchase
transaction_qty – Quantity purchased
unit_price – Price per item
store_location – Store location
product_category – Category of product
product_type – Type of product
product_detail – Specific product description

🛠️ **Tools & Technologies**
Miro (Project Planning)
Databricks SQL (Data querying and aggregation)
Excel (Data visualization and dashboards)
GitHub (Version control and project documentation)

📊 **Key Analysis Performed**
Revenue analysis by product category and type
Identification of peak sales hours
Trend analysis by day and month
Customer purchasing patterns
Best-selling products

📈 **Dashboard Insights**

The dashboard includes:
Total revenue by product type
Peak time intervals for sales
Quantity sold by category
Best-selling products
Visual charts and graphs for storytelling

💡 **Recommendations**

Based on the analysis:

Focus on promoting high-revenue products
Optimize staffing during peak hours
Introduce targeted promotions during low-sales periods
Expand popular product categories
Improve product placement and marketing strategies
